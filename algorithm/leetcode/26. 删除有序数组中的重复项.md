# 26. 删除有序数组中的重复项

**日期**：2026-07-01
**难度**：简单
**链接**：[LeetCode 26](https://leetcode.com/problems/remove-duplicates-from-sorted-array/)

## 题目描述

给定一个非严格递增排列的整数数组 `nums`，请你**原地**删除重复出现的元素，使每个元素只出现一次，返回删除后数组的新长度。元素的相对顺序应该保持一致。

由于在某些语言中不能改变数组的长度，所以必须将结果放在数组 `nums` 的第一部分。更规范地说，如果在删除重复项后有 `k` 个元素，那么 `nums` 的前 `k` 个元素应该保存最终结果。

## 解题思路

由于输入数组 `nums` 已经是**有序**的，重复的元素一定会相邻。我们可以使用**快慢双指针（Two Pointers）**法：
1. 定义一个慢指针 `slow`，指向当前已经处理好的无重复数组的末尾（初始为 0）。
2. 定义一个快指针 `fast`，遍历整个数组（从 1 开始）。
3. 当 `nums[fast] != nums[slow]` 时，说明找到了一个全新的唯一元素：
   - 慢指针向后移动一位（`slow++`）。
   - 将快指针指向的值复制到慢指针的位置（`nums[slow] = nums[fast]`）。
4. 遍历结束后，`slow + 1` 就是新数组的长度。

## 代码实现

```java
class Solution {
    public int removeDuplicates(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        
        int slow = 0;
        for (int fast = 1; fast < nums.length; fast++) {
            if (nums[fast] != nums[slow]) {
                slow++;
                nums[slow] = nums[fast];
            }
        }
        // 返回长度，所以是索引 + 1
        return slow + 1;
    }
}

## 复杂度分析

- **时间复杂度**：O(n)，其中n是数组的长度。快指针 fast 仅将数组从头到尾遍历了一次。
- **空间复杂度**：O(1)，没有使用额外的数组空间，只使用了两个指针变量，符合题目“原地（in-place）”修改的要求。

## 优化思路（进阶拓展）

- **拓展思考：如果题目要求“最多允许重复两次”（例如 LeetCode 80 题），该怎么做？
- **通解公式：快慢指针其实可以拓展到“允许重复 $k$ 次”的通用问题，核心逻辑是比较 nums[fast] 与 nums[slow - k + 1] 是否相同。

## 收获

1. 第一次接触了“原地修改”数组的题目，掌握了快慢指针在有序数组去重中的精妙应用。
2. 明白了不需要真的去“删除”数组元素，只需要把有效元素往前移动，覆盖掉无用元素即可。

# 27. 删除元素

**日期**：2026-07-01
**难度**：简单
**链接**：[LeetCode 27](https://leetcode.cn/problems/remove-element/)

## 题目描述

给你一个数组 `nums` 和一个值 `val`，你需要**原地**移除所有数值等于 `val` 的元素。元素的顺序可以改变。然后返回 `nums` 中与 `val` 不同的元素的数量。

## 解题思路

使用**快慢双指针**法：
1. 定义一个慢指针 `index`，用来记录下一个非 `val` 元素应该存放的位置（初始为 0）。
2. 使用快指针 `i` 遍历整个数组。如果当前元素 `nums[i]` 不等于 `val`，说明它是需要保留的元素，将其复制到 `nums[index]` 的位置，随后 `index` 向后移动一位。
3. 遍历结束后，`index` 的值正好就是新数组中不等于 `val` 的元素总个数。

## 代码实现

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int index = 0;
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] != val) {
                nums[index] = nums[i];
                index++;
            }
        }
        return index;
    }
}
# 复杂度分析

- **时间复杂度**：O(n)，其中n是数组 nums 的长度。整个数组只需要被单次遍历。
- **空间复杂度**：O(1)，直接在原数组上进行修改，没有开辟额外的内存空间。.

优化思路（待学习）
可以使用双指针碰撞法进行优化。让左右两个指针分别从数组两端向中间靠拢，当左指针遇到 val 时，直接将右指针指向的末尾元素复制过来，然后右指针左移。这样在要删除的元素很少时，可以大幅减少不必要的元素复制和移动次数。

## 收获

1. 第一次接触了快慢指针在数组原地修改问题中的应用
2. 明白了在内存连续的线性结构中，所谓的删除其实就是用后面的有效数据覆盖掉前面无效的数据。

# mall 项目环境搭建记录

**日期**：2026-07-01

## 环境清单

| 工具 | 版本 | 用途 |
|------|------|------|
| JDK | 17 (Temurin) | Java 运行环境 |
| Maven | 3.9.16 | 项目构建 |
| Docker | 29.5.3 | 容器化部署 |
| MySQL | 8.0.46 | 核心数据库 |
| IDEA | Community | IDE |

---

## 1. Docker 安装与配置

- 安装 Docker Desktop（默认安装即可）
- 修改镜像存储位置到 E 盘（避免 C 盘空间不足）
- 验证：`docker --version`

---

## 2. MySQL 本地安装 + 数据库创建

- 安装 MySQL 8.0（安装时记住 root 密码）
- 配置环境变量，确保 `mysql` 命令可用
- 创建 mall 数据库：

```sql
CREATE DATABASE mall DEFAULT CHARACTER SET utf8mb4;

- 导入 SQL 脚本：

```sql
USE mall; SOURCE E:/Projects/mall/document/sql/mall.sql;

- 验证：`SHOW TABLES;` 能看到 ums_admin、pms_product 等表

---

## 3. Redis Docker 启动

```bash
docker run -d --name redis -p 6379:6379 redis:latest

验证：`docker ps` 能看到 redis 容器在运行。

---

## 4. 项目导入 IDEA

- 从 Gitee 克隆 mall 项目：`git clone https://gitee.com/viryyp/mall.git`
- 项目移动到 E 盘：`E:\Projects\mall`
- IDEA 打开项目，等待 Maven 依赖下载完成
- 确认 Project SDK 为 JDK 17

---

## 5. 修改数据库配置

文件：`mall-admin/src/main/resources/application-dev.yml`

```yaml
spring: datasource: url: jdbc:mysql://localhost:3306/mall?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false username: root password: 你的密码 data: redis: host: localhost port: 6379

---

## 6. 启动成功验证

- 运行 `MallAdminApplication`
- 控制台出现：`Started MallAdminApplication in 5.825 seconds`
- 浏览器访问 `http://localhost:8080`
- 返回 401（暂未登录），说明服务正常运行

---

## 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| Docker 镜像下载慢 | 网络问题 |
| MySQL 端口冲突 | 本地 MySQL 和 Docker MySQL 同时占用 3306 | 停止 Docker MySQL，改用本地 MySQL |
| 本地 MySQL 服务名不对 | net stop MySQL80 失败 | 用 sc query 查找正确服务名，或直接 CMD 登录 |
